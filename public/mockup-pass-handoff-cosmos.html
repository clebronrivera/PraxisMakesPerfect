02 · MOTION

The handoff, in two beats

The path the user takes from anonymous to authenticated — and the one piece of state we carry across.

Beat 01 · Landing
Pass the Praxis. Become the psychologist.

Boot sequence (root → domains → skills) plays once per session. Hero CTA "Begin diagnostic" routes new users; "Sign in" routes returning users to /auth.

→/auth
Beat 02 · Login
Resume your study.

Reads localStorage.lastQueueState if present and surfaces it on the left panel. Successful auth → /today.

Behavior

  • Boot sequence plays once per session (gated on sessionStorage.pass-boot-seen). Skip button always visible bottom-right.
  • Starfield drifts at 30 px / 30 s; pauses on tab blur via visibilitychange.
  • Reduced motion · halts boot, brain pulse, starfield. CTA gradient becomes solid peach.

Edge cases

  • Returning user · if a valid session cookie exists, replace "Begin diagnostic" with "Resume your study →" and route to /today.
  • Slow font load · Fraunces falls back to Georgia, serif with -2 % letter-spacing offset to prevent reflow.
  • ≤ 768 px · brain figure stacks above H1; H1 drops to 56 px.

States to build

  • Idle · CTA disabled, fields empty.
  • Validating · CTA enabled, full-strength gradient.
  • Submitting · CTA shows pulse dots, fields lock.
  • Error · 1 px peach border on offending field; inline message Inter 12 px / peach below.
  • Locked · 5 failed attempts → CTA replaced by "Try again in 5:00", countdown live.

Edge cases & A11y

  • Google redirect · open in same tab; on return, hydrate from server-set cookie before rendering.
  • Autofill · respect; do not animate field state changes during autofill.
  • Tab order · Email → Password → Show → Keep-signed → Submit → Forgot → Diagnostic.
  • Contrast · all text on navy passes WCAG AA at 14 px+. Dim text (rgba(203,213,225,.7)) is for ≥ 14 px only.
  • 1280 px · stat strip becomes 2-across. ≤ 768 px stacks columns.

Animation timeline · 8 s loop

0.0 s → D1 ramp-up · 600 ms ease-out
2.0 s → D2 ramp-up · spark D1 → D2
4.0 s → D3 ramp-up · spark D2 → D3
6.0 s → D4 ramp-up · spark D3 → D4
8.0 s → loop · cross-fade trailing lobe to 0.35

Props

size · number · default 560
motion · bool · halts loop at prefers-reduced-motion
live · bool · binds lobe alpha to current per-domain mastery
onLobeHover · (domainId) ⇒ void · used for product-surface tooltips

Implementation notes

  • SVG only · no canvas, no images. Bezier paths defined in brain-activation.tsx.
  • Glow · radial gradient + 6 px Gaussian blur filter on a duplicate path; not a CSS box-shadow.
  • Sparks · production component · single <circle> animated along path via SVG animateMotion. One sprite per spark, recycled.
  • Sparks · this static mockup · uses CSS offset-path as a visual equivalent — declarative and well-supported in modern browsers, but lacks the recycled-sprite efficiency the React version should have.
  • Reduced motion · all 4 lobes hold at 0.7 alpha; sparks suppressed.

Where it appears

  • Landing · hero, decorative · 560 px · live=false.
  • Today / Home · sidebar · 96 px · live=true (lobes glow brighter when that domain's mastery is up).
  • Diagnostic results · alongside the per-skill grid · 280 px · live=true with onLobeHover filtering the table.
  • Login left panel (optional) · 200 px static · used as a watermark behind the engine voice only if marketing wants it.
05 · COMPONENT

Primary CTA · states

The pill button is the only place the CTA gradient appears as a fill. Six states; spec applies to both screens.

Idle
Default. 14 / 24 padding · pill 999 · gradient at full strength. Inter 600 / 14 px ink #1E1B3A.
Hover
Lift 1 px · brightness 1.04 · halo shadow 0 8 28 rgba(252,213,180,.25). 120 ms ease-out.
Disabled
Drop gradient. Hair fill. Ink at 40 % opacity. Cursor not-allowed.
Loading
Lock width to idle measurement; replace label with three dots pulsing 0.8 s. Field inputs disabled.
Error retry
Tinted miss-color background. Border rgba(252,165,165,.4). Used after auth failure.
Locked
After 5 failed attempts. Mono 13 px · countdown live. Fields read-only.
06 · ENGINEERING

Implementation notes

Endpoints expected

  • POST /api/auth/session · email + password → 200 + cookie, 401 invalid, 423 locked.
  • GET /api/auth/sso/{provider} · 302 to provider; on return, server sets cookie.
  • GET /api/user/state · returns { firstName, daysToTest, streak, mastery, dailyDoseMinutes } for the left-panel stat strip.

Analytics events

  • landing.cta.diagnostic.click
  • landing.nav.signin.click
  • login.sso.click · {provider}
  • login.submit · {success:bool, errorCode?:string}
  • login.locked · after 5th failure
