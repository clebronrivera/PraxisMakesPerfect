Design reference

AI Grounding Rules

How the model is constrained to produce a focused, accurate plan — not a hallucinated one.

What Claude receives (hardcoded inputs)
Pre-computed skill data
All 45 skills × status label, confidence-weighted accuracy, trend, distractor flag (L1/L2/L3), urgency score
Priority clusters
Pre-grouped by gap type: misconception, unlearned, unstable, near-mastery. Claude receives the groups, not raw scores.
Time budget + session frame
Available hours/week × weeks remaining = total study hours. Session length and frequency are pre-calculated — Claude cannot change them.
Skill content metadata
Vocabulary terms, misconception text, case archetypes, laws & frameworks — pulled from static metadata files, not inferred by Claude.
User intake answers
Depth, focus scope, exam date, stated challenges, goals — passed verbatim as context parameters.
Onboarding profile
Role, training stage, weekly availability, biggest challenges — read from the user's saved profile, not re-asked.
Hard constraints in the prompt
✕ Cannot add sections beyond the depth template (Quick = 3, Standard = 5, Comprehensive = 9). Section list is injected as a strict schema.
✕ Each section has a word budget (e.g., Readiness Snapshot = 180 words max). Overrun = truncated in post-processing before PDF generation.
✕ Cannot recommend external resources, websites, books, or third-party tools. Prompt explicitly instructs: "All recommendations must use only the platform's question bank and learning path."
✕ Cannot invent skill names, exam dates, scores, or user data. All factual fields are slot-filled by code before the prompt is sent.
✕ Cannot change session counts, durations, or weekly targets — pre-set by the preprocessor. Claude writes around the schedule, not into it.
