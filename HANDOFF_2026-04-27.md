## Strategic context (read once)

Two paths were considered:

**Path A — PMP-in-place refactor.** Evolve PraxisMakesPerfect from single-exam to multi-exam in place via a 12-phase plan. ~4–8 weeks. Lives with PMP's scars (`legacySkillIdMap` translate-on-read, vestigial `primary_exam`, RLS designed for one test, hardcoded marketing copy).

**Path B — Greenfield PASS V1.** Build new platform from scratch. Lift patterns from PMP, reimplement clean against multi-exam abstractions from day one. ~3–6 months for feature parity. PMP keeps running for current Praxis users until PASS launches.

**Decision: Path B (greenfield).** Operator's reason: "If we're not able to isolate certain parts of this codebase and we continue drift or continue bringing old crap over then I fear that it'll never be clean."

**Sequencing decision:** All 11 Keystone exams must be authoring-complete before PASS Phase 1+. PASS Phase 0 (stack + scaffolding) can start immediately and runs in parallel with Keystone authoring. Phase 1+ wait at the Keystone gate.

**PMP user fate:** Decision deferred — tracked as `P-USER-MIGRATION` in `PASS V1/docs/plans/pass-platform/BLOCKERS.md`. Not blocking until PASS Phase 11.

## Things NOT to do

- **Do not work in PASS V1 without following the OS at `docs/plans/pass-platform/`.** Random scaffolding edits will drift the state machine.
- **Do not commit refactor work to PMP main.** PMP stays on its current single-exam path until PASS reaches feature parity.
- **Do not begin PASS Phase 1 before the Keystone gate.** Phase 0 → Phase 1 transition is gated by operator confirmation that all 11 Keystone exams are authoring-complete.
- **Do not copy PMP files wholesale into PASS V1.** Patterns are lifted (the architecture); files are reimplemented (the code). See PASS V1 `DECISIONS.md` #2.
- **Do not introduce Stripe / billing into PASS V1 before Phase 11.** v1 ships without paywall per `DECISIONS.md` #12.
- **Do not author Spanish translations of English exams.** PCMAS is Spanish-native; the other 10 are English-only. No translation layer.
- **Do not mark PMP-side OS files as "active."** They are reference material per the greenfield decision.
