# Phase C — error_cluster_tag Glossary

This is the shared tag registry for all content authoring contributors.

**Rules:**
1. Check this list before inventing a new tag. Prefer an existing tag when semantically equivalent.
2. Only add a new tag when no existing tag fits. New tags require: one-line definition + two example questions listed here before first use.
3. Tags must be lowercase, hyphenated, 1–4 words.
4. Run a per-skill tag normalization check before final handoff — verify all tags used within a skill are in this glossary and consistent.

---

## Active Tags

| Tag | Definition | Example Skills |
|-----|-----------|----------------|
| `model-conflation` | Student confuses one named model, framework, or approach with another that has a different goal or scope | CON-01 (behavioral vs. consultee-centered), DBD-03 (assessment models) |
| `scope-overgeneralization` | Student applies a rule, law, or principle more broadly than it actually applies | LEG-01 (FERPA universality error), ETH-01 (ethical obligations) |
| `scope-undergeneralization` | Student applies a rule, law, or principle too narrowly — misses cases it actually covers | LEG-02 (IDEA eligibility), ACA-08 (special ed law) |
| `sequence-inversion` | Student reverses the correct order of steps in a procedure or process | DBD-01 (referral sequence), MBH-03 (FBA → placement order) |
| `component-confusion` | Student places a concept, tool, or step inside the wrong category, process, or system | MBH-03 (IQ test as FBA component), DBD-03 (assessment type mismatch) |
| `indirect-direct-confusion` | Student confuses an indirect service delivery mode (through a teacher/parent) with direct service to the student | CON-01, CON-02, MBH-02 |
| `purpose-confusion` | Student knows a tool, law, or model exists but misidentifies what it is for | LEG-03 (504 vs. IDEA purpose), SAF-02 (risk assessment purpose) |
| `prerequisite-skipping` | Student jumps to a later step before completing required earlier steps | CON-01 (referral before consultation), DBD-01 (intervention before identification) |
| `label-retrieval` | Student understands the concept but cannot retrieve or match the correct named term, law, or model | LEG-01, LEG-02, DBD-06 |
| `overgeneralization` | General pattern where student applies a rule or concept too broadly (use when scope-overgeneralization is too specific) | Various |
| `population-confusion` | Student applies criteria or procedures designed for one population to a different population | ACA-07 (LD criteria), DIV-01 (assessment population misapplication) |
| `role-confusion` | Student assigns a responsibility or action to the wrong professional role | ETH-03 (supervisor vs. supervisee), CON-01 (consultant vs. consultee) |
| `causation-correlation` | Student conflates correlation with causation, or assumes direction of effect | RES-02, RES-03 |
| `validity-reliability-confusion` | Student confuses validity and reliability concepts or misapplies them | DBD-02, DBD-10 |
| `norm-criterion-confusion` | Student confuses norm-referenced and criterion-referenced interpretation | DBD-06, DBD-02 |
| `tier-level-confusion` | Student applies a MTSS/RTI tier's intervention type to the wrong tier level | SWP-01, SWP-02, DBD-08 |
| `eligibility-criteria-confusion` | Student misidentifies which criteria or conditions trigger a legal eligibility determination | LEG-02, ACA-07, ACA-08 |
| `consent-confidentiality-confusion` | Student confuses who holds consent rights, what confidentiality protects, or when disclosure is required | LEG-01, LEG-04, ETH-01 |
| `developmental-stage-mismatch` | Student applies knowledge from one developmental stage to a different stage inappropriately | DEV-01, MBH-01 |
| `treatment-assessment-confusion` | Student conflates an assessment process with an intervention/treatment process | MBH-01, DBD-05, MBH-03 |
